# Example: Launched Project

## Prompt

```text
before you build: We launched an AI resume checker. We get traffic, but almost nobody pays. Should we add more templates?
```

## What The Skill Should Focus On

- Whether the problem is traffic, activation, retention, trust, or payment.
- Whether more features would avoid the real bottleneck.
- Whether users trust the output enough to pay.
- Whether the pricing model matches usage frequency.

## Good Output Shape

```markdown
## Project Reality Check

Current situation:
- The product is launched, has traffic, but weak payment.

Biggest risk:
- This is likely a willingness-to-pay or trust problem, not a template-count problem.

Do not rush into more features:
- More templates will not fix weak payment if users do not trust the advice or see enough career impact.

Validate next:
1. Interview 10 visitors who used the checker but did not pay.
2. Test a paid expert review or paid rewrite before adding automation.
3. Compare conversion between "AI score" and "get 3 specific fixes for this job posting."

Recommendation:
Validate first
```
